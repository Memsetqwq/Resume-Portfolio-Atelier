# 动效骨架 / Motion Skeleton (v1.1.4)

> 履历裁缝出品的所有 HTML 默认套用的**统一动效系统**。
> 解决 5 个 style template 各自实现动效 → 表现参差、动效缺失的问题。
>
> 跟 SOUL.md "关于动效(底线原则)" + `docs/效果要素指南.md` 配合使用。

---

## 一句话

**2 个 baseline 必装 + ≤ 3 个 adaptive 按用户特点启用** —— 所有 HTML 出厂默认有动效,但不超"克制"红线。

---

## 模型(2 + 3)

### Baseline 1:滚动渐显(`IntersectionObserver`)

- **效果**:元素进入视口 100px 内时,从 opacity 0 + transform → opacity 1 + transform none
- **时长**:`var(--motion-base)` 默认 400ms(赛博未来覆盖为 200ms)
- **曲线**:`var(--ease-out)` = `cubic-bezier(0.16, 1, 0.3, 1)`
- **触发元素**:`.reveal` 类(每个 section + 每段经历 + 部分装饰元素)
- **降级**:
  - 关 JS → `.no-js .reveal` 默认 opacity 1
  - `prefers-reduced-motion: reduce` → transition none,直接显示
  - IntersectionObserver 不支持 → 全部直接 visible
- **per-style 差异化**(写在各 template 自己的 CSS):
  - 极简留白:translateY(8px),400ms
  - 杂志编辑:translateY(12px),400ms
  - 赛博未来:translateY(4px),**200ms**(局部覆盖 `--motion-base`)
  - 手账拼贴:scale(0.95)+rotate(1deg),300ms

### Baseline 2:链接/装饰 hover 微反馈

- **效果**:链接颜色 + 下划线 / 卡片边框 / 装饰元素 鼠标悬停时平滑过渡
- **时长**:`var(--motion-fast)` 默认 150ms
- **曲线**:`ease`(默认浏览器曲线,够快不拖沓)
- **触发元素**:所有 `<a>` + 带 `.hover-lift` 类的装饰元素
- **降级**:无 JS 依赖,纯 CSS,任何环境都生效
- **per-style 差异化**:
  - 极简留白:border-bottom-color 透明 ↔ accent
  - 杂志编辑:border-bottom-color 灰 ↔ 米白
  - 赛博未来:**色 + border 同时变**(绿色 ↔ 紫色,带 dashed border)
  - 手账拼贴:背景色 + 微微 scale(1.02)

### Adaptive 1:数字递增(`count-up`)

- **效果**:`.metric-num` / `.skill-pct` 元素从 0 在 800ms 内递增到目标值,数字带 `ease-out cubic` 曲线
- **触发**:`[data-countup="18"]` 属性,主 Agent 按用户内容决定加在哪
- **触发条件**(主 Agent 决策):
  - 硬数字 metric card ≥ 3 个 → 必开
  - 1-2 个 → 可选(用户想要"再活泼点"就开)
  - 0 个 → 不开(没元素可作用)
- **降级**:
  - `prefers-reduced-motion: reduce` → 直接显示最终值,不递增
  - 打印 PDF → `@media print` 里 transition 关,显示最终值
  - JS 失败 → 显示原始 HTML 文本(就是 `{{metric_1_num}}` 占位符值)
- **注意点**:
  - `data-countup` 只存**纯数字**(如 `18` `0.81` `800`),前缀 `+` 后缀 `%` 用 `<span>` 包裹
  - 数字精度:`parseFloat` 后保留原始小数位(0.81 显示为 0.81)

### Adaptive 2:Hero stagger 入场

- **效果**:首屏容器内的 direct children 依次淡入,每 100ms 一个(手账拼贴 150ms)
- **触发**:`[data-stagger]` 属性在 `.header` / `.masthead` / `.prompt-line + h1` 等首屏容器上
- **触发条件**(主 Agent 决策):
  - 包装区间 40%+ → 必开(求职者想要"冲一把",首屏要亮)
  - 设计师 / 创意岗 / freelance 接单 → 必开(气质更活泼)
  - 包装区间 20% + 应届求职 → 默认开(中性)
  - 工程师 / 科研 / 留学 CV → 不开(克制气质优先)
- **降级**:
  - `prefers-reduced-motion: reduce` → 直接显示
  - 关 JS → `[data-stagger] > *` 默认 opacity 1
- **per-style 差异化**:
  - 极简留白:stagger 100ms 间隔,header 内 3 个子元素(name/title/contact)
  - 杂志编辑:stagger 150ms 间隔(略慢,editorial 节奏)
  - 赛博未来:stagger 80ms 间隔(快,"终端启动"感)
  - 手账拼贴:stagger **150ms 间隔**,带 -1deg rotation(慢,"贴纸一张张贴上"感)

### Adaptive 3:章节标号 + 装饰元素 accent line drawing

- **效果**:h2 章节标号旁的 accent 横线宽度从 0 → 32px 渐进绘出;技能条宽度从 0 → 百分比扩展;装饰元素(◆ ▲ ▸ ✿)在 hover 时切换颜色或位移
- **触发**:CSS transition + IntersectionObserver 联动(技能条进入视口时 trigger)
- **触发条件**(主 Agent 决策):
  - h2 章节标号:99% 必开(已是 template 默认,没有理由关)
  - 技能条扩展:有 `.skill-bar` / `.level-bar` 时必开
  - 装饰元素 hover:99% 必开
- **降级**:
  - `prefers-reduced-motion: reduce` → transition none
  - 打印 → 直接显示完整状态
- **per-style 差异化**:
  - 极简留白:h2 短横线 32px,ease-out 400ms
  - 杂志编辑:name 下划线 64px → 120px 渐进
  - 赛博未来:技能条水平 bar 扩展 + 数字 count-up(组合)
  - 手账拼贴:贴纸卡 hover 旋转加深 0.5°

## 共享 Token(各 template 都写到 `:root`)

```css
:root {
  --motion-fast: 150ms;      /* hover 微反馈 */
  --motion-base: 400ms;      /* 滚动渐显 */
  --motion-slow: 800ms;      /* count-up, stagger */
  --ease-out: cubic-bezier(0.16, 1, 0.3, 1);
}
```

**局部覆盖示例**(赛博未来用更快的 base):

```css
:root {
  --motion-base: 200ms;      /* 局部覆盖,reveal 更快 */
}
```

## 共享脚本(各 template 都加载)

```js
(function () {
  'use strict';
  const REDUCED_MOTION = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

  // === 1. Scroll reveal ===
  function initReveal() {
    if ('IntersectionObserver' in window) {
      const io = new IntersectionObserver((entries) => {
        entries.forEach((e) => {
          if (e.isIntersecting) { e.target.classList.add('visible'); io.unobserve(e.target); }
        });
      }, { threshold: 0.1 });
      document.querySelectorAll('.reveal').forEach((el) => io.observe(el));
    } else {
      document.querySelectorAll('.reveal').forEach((el) => el.classList.add('visible'));
    }
  }

  // === 2. Count-up (数字递增) ===
  function countUp(el, target, duration) {
    if (REDUCED_MOTION) return;
    duration = duration || 800;
    const start = performance.now();
    const isFloat = target % 1 !== 0;
    function tick(now) {
      const elapsed = now - start;
      const progress = Math.min(elapsed / duration, 1);
      const eased = 1 - Math.pow(1 - progress, 3);  // ease-out cubic
      const current = target * eased;
      el.textContent = isFloat ? current.toFixed(2) : Math.round(current);
      if (progress < 1) requestAnimationFrame(tick);
      else el.textContent = isFloat ? target.toFixed(2) : target;  // 最终值兜底
    }
    requestAnimationFrame(tick);
  }

  // === 3. Hero stagger ===
  function initStagger() {
    if (REDUCED_MOTION) return;
    document.querySelectorAll('[data-stagger]').forEach((el) => {
      // 触发下一次渲染,加 visible class 走 CSS 过渡
      requestAnimationFrame(() => {
        setTimeout(() => el.classList.add('staggered'), 50);
      });
    });
  }

  // === Init ===
  document.body.classList.remove('no-js');
  initReveal();
  document.querySelectorAll('[data-countup]').forEach((el) => {
    const target = parseFloat(el.dataset.countup);
    if (!isNaN(target)) countUp(el, target);
  });
  initStagger();
})();
```

**配套 CSS**(各 template 也需要):

```css
/* 关 JS 默认 visible */
.no-js .reveal { opacity: 1; transform: none; }
.no-js [data-stagger] > * { opacity: 1; transform: none; }

/* Reduced motion 全关 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    transition-duration: 0.01ms !important;
    animation-duration: 0.01ms !important;
  }
  .reveal, [data-stagger] > * { opacity: 1 !important; transform: none !important; }
}

/* Hero stagger 默认态(被 .staggered 触发) */
@media (prefers-reduced-motion: no-preference) {
  [data-stagger] > * {
    opacity: 0;
    transform: translateY(8px);
    transition: opacity var(--motion-base) var(--ease-out),
                transform var(--motion-base) var(--ease-out);
  }
  [data-stagger].staggered > * { opacity: 1; transform: none; }
  [data-stagger].staggered > *:nth-child(1) { transition-delay: 0ms; }
  [data-stagger].staggered > *:nth-child(2) { transition-delay: 100ms; }
  [data-stagger].staggered > *:nth-child(3) { transition-delay: 200ms; }
  [data-stagger].staggered > *:nth-child(4) { transition-delay: 300ms; }
}
```

| 资深求职 / 20% | ✓ | ✓ | metric ≥ 2 必开 | 默认开 | ✓ |

## 与 SOUL 动效红线的协调

原 SOUL 规则:**默认每个 HTML 最多 2-3 个动效**(多了就俗)

新规则:**2 baseline(必装,关 JS 也起作用) + ≤ 3 adaptive(主 Agent 按用户特点自适应)** —— 总数 ≤ 5 个

> 为什么放宽到 5:**2 baseline 都是"通用"动效**(滚动 + hover),不占用户注意力;真正"显示"的动效只有 1-3 个 adaptive,所以"主观动效数"还是 ≤ 4。Baseline 是"骨架",Adaptive 是"显示"。
>
> 这条规则**高于所有风格偏好** —— 用户说"我想要动效多"也不行,SOUL 底线是"读得清 > 动效多"。

## 视觉亮点(非动效,但跟动效协同)

> 用户反馈 v1.1.4:`你这默认效果太单调朴素了`。
> 回应:除了动效骨架外,在视觉锚点上做了增强。两件事协同:
> 1. **动效骨架**(本文件 2+3 模型)— 时间维度的丰富
> 2. **视觉锚点**(各 template 静态加强)— 空间维度的丰富

### 各 template 视觉锚点清单

| Style | 视觉亮点 |
|-------|---------|
| 极简留白 | hero 大装饰 "00"(右上角)+ ◆ 角标 + ◆/▲/▸/─ 装饰符 + section-divider + 卡片 hover 上抬 |
| 杂志编辑 | hero 大装饰 "00"(右上角,罗马数字)+ ISSUE NO. 角标 + CHAPTER Ⅰ Ⅱ 罗马数字 + chapter-deck 双层标 + ¶ 联系符 + 作品灰度→彩色 + FIG. 编号章 |
| 赛博未来 | hero ASCII 装饰 + status bar(online 点 + UTC)+ status dot pulse + 命令行分块 + glitch hover on h1 + 技能条渐变 + 状态徽章(badge) + terminal exit footer |
| 手账拼贴 | hero 日期贴纸(右侧圆形手写)+ intro-tag 圆角胶囊 + handwritten-underline SVG + 章节 chapter-mark 圆圈 + 心情标签 ✿ + work-thumb 占位 + skill-tag 角度错落 + footer 圆形印章 |

### 主 Agent 出 v1 时查视觉锚点的决策

- template 已自带锚点 → 不主动加额外元素
- 用户特殊要求(摄影师/建筑师等)→ 加专属 visual mark(占位图 + alt 文案)
- 不要把所有锚点都开 → 选 3-5 个跟用户气质匹配的启用

## 验证清单(每个 template 必跑)

- [ ] `:root` 有 `--motion-fast` / `--motion-base` / `--motion-slow` / `--ease-out`
- [ ] `.reveal` 的 transition 用 token 而非写死 `0.4s`
- [ ] `[data-stagger]` CSS 在 template 里(per-style 间隔可能不同)
- [ ] 共享脚本(↑ 上面那段)在 template 里
- [ ] `prefers-reduced-motion: reduce` 时 transition 全 0.01ms
- [ ] `.no-js .reveal` 默认 visible
- [ ] 移动端 640px 以下 reveal 距离减半(`translateY(4px)` 等)
- [ ] `@media print` 时 reveal / stagger 全部强制 visible + transition 0
- [ ] 视觉锚点跟 template 气质匹配(不是把所有锚点都用上)
