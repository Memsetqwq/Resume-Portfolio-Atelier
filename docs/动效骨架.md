# 动效骨架 / Motion Skeleton (v1.1.4)

> 履历裁缝出品的所有 HTML 默认套用的**统一动效系统**。
> 解决 5 个 style template 各自实现动效 → 表现参差、动效缺失的问题。
>
> 跟 SOUL.md "关于动效(底线原则)" + `docs/效果要素指南.md` 配合使用。

---

## 一句话

**2 个 baseline 必装 + ≤ 5 个 adaptive 按用户特点启用 + 动效全覆盖 + 双向触发** —— 所有 HTML 出厂默认有动效,容器 / 子元素 / 装饰元素都有进入动画,无死角;**上下反复滑动都会重放**入行动画,不是"看过一次就静默"。

---

## 模型(2 + 5 + 双向触发)

### Baseline 1:滚动渐显(`IntersectionObserver`,双向触发)

- **效果**:元素进入视口 100px 内时,从 opacity 0 + transform → opacity 1 + transform none;**离开视口时移除 `visible` class,下次再进入时重放**
- **时长**:`var(--motion-base)` 默认 400ms(赛博未来覆盖为 200ms)
- **曲线**:`var(--ease-out)` = `cubic-bezier(0.16, 1, 0.3, 1)`
- **触发元素**:`.reveal` 类(每个 section + 每段经历 + 部分装饰元素)
- **双向触发(v1.1.4 加强版)**:
  - 进入视口 → 加 `.visible`(走 CSS transition 入场)
  - 离开视口 → 移除 `.visible`(回到初始态,下次进入再触发)
  - IntersectionObserver **不调用 `unobserve`**,持续观察
  - threshold `0.12` + rootMargin `-8%` 防快速划过误触发
- **降级**:
  - 关 JS → `.no-js .reveal` 默认 opacity 1
  - `prefers-reduced-motion: reduce` → transition none,直接显示,**不重复触发**
  - IntersectionObserver 不支持 → 全部直接 visible(一次性,不重放)
- **per-style 差异化**(写在各 template 自己的 CSS):
  - 极简留白:translateY(8px),400ms
  - 杂志编辑:translateY(12px),400ms
  - 赛博未来:translateY(4px),**200ms**(局部覆盖 `--motion-base`)
  - 手账拼贴:scale(0.95)+rotate(1deg),300ms

### Baseline 2:链接/装饰 hover 微反馈

- **效果**:链接颜色 + 下划线 / 卡片边框 / 装饰元素 鼠标悬停时平滑过渡
- **时长**:`var(--motion-fast)` 默认 150ms
- **曲线**:`ease`(默认浏览器曲线,够快不拖沓)
- **触发元素**:所有 `<a>` + 带 `.hover-lift` 类的装饰元素
- **降级**:无 JS 依赖,纯 CSS,任何环境都生效
- **per-style 差异化**:
  - 极简留白:border-bottom-color 透明 ↔ accent
  - 杂志编辑:border-bottom-color 灰 ↔ 米白
  - 赛博未来:**色 + border 同时变**(绿色 ↔ 紫色,带 dashed border)
  - 手账拼贴:背景色 + 微微 scale(1.02)

### Adaptive 1:数字递增(`count-up`,双向触发)

- **效果**:`.metric-num` / `.skill-pct` 元素从 0 在 800ms 内递增到目标值,数字带 `ease-out cubic` 曲线
- **触发**:`[data-countup="18"]` 属性,主 Agent 按用户内容决定加在哪
- **双向触发(v1.1.4 加强版)**:
  - 进入视口 → 归 0 → 递增到目标值(count-up 重跑)
  - 离开视口 → 归 0(下次进入再递增)
  - IntersectionObserver **不调用 `unobserve`**,持续观察
  - threshold `0.5`(数字至少 50% 进入视口才触发,避免误触发)
- **触发条件**(主 Agent 决策):
  - 硬数字 metric card ≥ 3 个 → 必开
  - 1-2 个 → 可选(用户想要"再活泼点"就开)
  - 0 个 → 不开(没元素可作用)
- **降级**:
  - `prefers-reduced-motion: reduce` → 直接显示最终值,不递增,不重放
  - 打印 PDF → `@media print` 里 transition 关,显示最终值
  - JS 失败 → 显示原始 HTML 文本(就是 `{{metric_1_num}}` 占位符值)
- **注意点**:
  - `data-countup` 只存**纯数字**(如 `18` `0.81` `800`),前缀 `+` 后缀 `%` 用 `<span>` 包裹
  - 数字精度:`parseFloat` 后保留原始小数位(0.81 显示为 0.81)

### Adaptive 2:Hero stagger 入场(双向触发)

- **效果**:首屏容器内的 direct children 依次淡入,每 100ms 一个(手账拼贴 150ms)
- **触发**:`[data-stagger]` 属性在 `.header` / `.masthead` / `.prompt-line + h1` 等首屏容器上
- **双向触发(v1.1.4 加强版)**:
  - 进入视口 → 移除 `staggered` → 下一帧重新加 `staggered`(CSS 重新跑 stagger 动画)
  - 离开视口 → 移除 `staggered`(下次进入再 stagger)
  - IntersectionObserver **不调用 `unobserve`**,持续观察
  - threshold `0.1`
- **触发条件**(主 Agent 决策):
  - 包装区间 40%+ → 必开(求职者想要"冲一把",首屏要亮)
  - 设计师 / 创意岗 / freelance 接单 → 必开(气质更活泼)
  - 包装区间 20% + 应届求职 → 默认开(中性)
  - 工程师 / 科研 / 留学 CV → 不开(克制气质优先)
- **降级**:
  - `prefers-reduced-motion: reduce` → 直接显示,不重放
  - 关 JS → `[data-stagger] > *` 默认 opacity 1
- **per-style 差异化**:
  - 极简留白:stagger 100ms 间隔,header 内 3 个子元素(name/title/contact)
  - 杂志编辑:stagger 150ms 间隔(略慢,editorial 节奏)
  - 赛博未来:stagger 80ms 间隔(快,"终端启动"感)
  - 手账拼贴:stagger **150ms 间隔**,带 -1deg rotation(慢,"贴纸一张张贴上"感)

### Adaptive 3:章节标号 + 装饰元素 accent line drawing(双向联动)

- **效果**:h2 章节标号旁的 accent 横线宽度从 0 → 32px 渐进绘出;技能条宽度从 0 → 百分比扩展;装饰元素(◆ ▲ ▸ ✿)在 hover 时切换颜色或位移
- **触发**:CSS transition + IntersectionObserver 联动(进入视口触发,离开复位 → 跟 Baseline 1 复用同一个 IO 实例)
- **触发条件**(主 Agent 决策):
  - h2 章节标号:99% 必开(已是 template 默认,没有理由关)
  - 技能条扩展:有 `.skill-bar` / `.level-bar` 时必开
  - 装饰元素 hover:99% 必开
- **降级**:
  - `prefers-reduced-motion: reduce` → transition none
  - 打印 → 直接显示完整状态
- **per-style 差异化**:
  - 极简留白:h2 短横线 32px,ease-out 400ms
  - 杂志编辑:name 下划线 64px → 120px 渐进
  - 赛博未来:技能条水平 bar 扩展 + 数字 count-up(组合)
  - 手账拼贴:贴纸卡 hover 旋转加深 0.5°

## Adaptive 4(动效全覆盖层):子元素 stagger reveal(双向联动)

> 这是 v1.1.4 加强后的"动效全覆盖"层 —— **不只是每个 section 进入视口时整个容器淡入,容器内部的每张卡片 / 每个标签 / 每个条目都依次进入**。**双向:容器进入视口时子元素 stagger 进入,容器离开时子元素一起复位**。

- **效果**:标记 `[data-reveal-group]` 的容器,其直接子元素进入视口时依次淡入(60-80ms 间隔,per-style 调整)
- **触发**:容器有 `.reveal` + `[data-reveal-group]` 双属性,IntersectionObserver 触发容器 `.visible`,子元素用 CSS `:nth-child` transition-delay 错开
- **双向联动**:容器 `.visible` class 被 Baseline 1 IO 加上时,子元素走 stagger 入场;容器 `.visible` 被移除时,子元素回到初始态(opacity 0 + transform),下次进入再 stagger
- **适用元素**(默认全开):
  - `.metrics-row > .metric`(每张 metric card 单独 stagger)
  - `.section > .exp-card`(每段经历卡片单独 stagger)
  - `.section > .project-card`(每个项目卡片单独 stagger)
  - `.skills-grid > .skill-tag`(每个技能标签单独 stagger)
  - `.footer > span`(footer 各部分单独 stagger)
  - `.contact > span/p`(联系信息各部分单独 stagger)
  - `.work` 卡片(杂志编辑作品)
  - `.item` 行(赛博未来 经历/项目)
- **降级**:
  - `prefers-reduced-motion: reduce` → 直接显示,无错开
  - 关 JS → 全部默认显示
  - 打印 → 直接显示
- **per-style 差异化**(错开间隔):
  - 极简留白:60ms
  - 杂志编辑:80ms(略慢,editorial 节奏)
  - 赛博未来:50ms(快,"终端逐行打印"感)
  - 手账拼贴:80ms(慢,"贴纸一张张贴上"感)

## 共享脚本(各 template 都加载,v1.1.4 加强版:双向触发)

```js
(function () {
  'use strict';
  const REDUCED_MOTION = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

  // === 1. Scroll reveal(双向触发)== =
  function initReveal() {
    if ('IntersectionObserver' in window) {
      const io = new IntersectionObserver((entries) => {
        entries.forEach((e) => {
          if (e.isIntersecting) {
            e.target.classList.add('visible');
            // h2 还触发 accent line drawing
            if (e.target.tagName === 'H2') e.target.classList.add('revealed');
          } else {
            // 离开视口:移除,下次再进入时重新触发动画
            e.target.classList.remove('visible');
            if (e.target.tagName === 'H2') e.target.classList.remove('revealed');
          }
        });
      }, { threshold: 0.12, rootMargin: '0px 0px -8% 0px' });
      document.querySelectorAll('.reveal, h2').forEach((el) => io.observe(el));
    } else {
      document.querySelectorAll('.reveal, h2').forEach((el) => {
        el.classList.add('visible');
        if (el.tagName === 'H2') el.classList.add('revealed');
      });
    }
  }

  // === 2. Count-up(双向触发)== =
  function countUp(el, target, duration) {
    if (REDUCED_MOTION) return;
    duration = duration || 800;
    const start = performance.now();
    const isFloat = target % 1 !== 0;
    function tick(now) {
      const elapsed = now - start;
      const progress = Math.min(elapsed / duration, 1);
      const eased = 1 - Math.pow(1 - progress, 3);
      const current = target * eased;
      el.textContent = isFloat ? current.toFixed(2) : Math.round(current);
      if (progress < 1) requestAnimationFrame(tick);
      else el.textContent = isFloat ? target.toFixed(2) : target;
    }
    requestAnimationFrame(tick);
  }
  if (!REDUCED_MOTION && 'IntersectionObserver' in window) {
    const countObserver = new IntersectionObserver((entries) => {
      entries.forEach((e) => {
        const el = e.target;
        const target = parseFloat(el.dataset.countup);
        if (isNaN(target)) return;
        if (e.isIntersecting) {
          el.textContent = '0';
          countUp(el, target);
        } else {
          el.textContent = '0';
        }
      });
    }, { threshold: 0.5 });
    document.querySelectorAll('[data-countup]').forEach((el) => countObserver.observe(el));
  } else if (!REDUCED_MOTION) {
    document.querySelectorAll('[data-countup]').forEach((el) => {
      const target = parseFloat(el.dataset.countup);
      if (!isNaN(target)) countUp(el, target);
    });
  } else {
    document.querySelectorAll('[data-countup]').forEach((el) => {
      el.textContent = el.dataset.countup;
    });
  }

  // === 3. Hero stagger(双向触发)== =
  function initStagger() {
    if (!REDUCED_MOTION && 'IntersectionObserver' in window) {
      const staggerObserver = new IntersectionObserver((entries) => {
        entries.forEach((e) => {
          const el = e.target;
          if (e.isIntersecting) {
            el.classList.remove('staggered');
            requestAnimationFrame(() => {
              setTimeout(() => el.classList.add('staggered'), 60);
            });
          } else {
            el.classList.remove('staggered');
          }
        });
      }, { threshold: 0.1 });
      document.querySelectorAll('[data-stagger]').forEach((el) => staggerObserver.observe(el));
    } else {
      document.querySelectorAll('[data-stagger]').forEach((el) => el.classList.add('staggered'));
    }
  }

  // === Init ===
  document.body.classList.remove('no-js');
  initReveal();
  initStagger();
})();
```

```css
/* 关 JS 默认 visible */
.no-js .reveal { opacity: 1; transform: none; }
.no-js [data-stagger] > * { opacity: 1; transform: none; }

/* Reduced motion 全关(不重放) */

  *, *::before, *::after {
    transition-duration: 0.01ms !important;
```

## 与 SOUL 动效红线的协调

原 SOUL 规则:**默认每个 HTML 最多 2-3 个动效**(多了就俗)

新规则:**2 baseline(必装,关 JS 也起作用) + ≤ 5 adaptive(主 Agent 按用户特点自适应) + 全部双向触发** —— 总数 ≤ 7 个,**但用户上下反复滑动时每个都会重放**

> 为什么放宽到 7:**2 baseline + Adaptive 1-3 都是"通用"动效**(滚动 + hover + 数字 + stagger + accent),不占用户注意力;真正"显示"的动效只有 Adaptive 4-5(子元素 stagger + 装饰元素微动),加起来仍克制。
>
> **动效全覆盖不等于"动效多到闪"** —— 视觉上仍是"骨架 + 微动"的感觉,不是"炫技"。只是**没有任何一个 part 是死气沉沉不动**的。
>
> **双向触发不等于"卡顿"** —— IntersectionObserver 持续观察 + 阈值防误触发,重放只在元素真正进入/离开视口时发生,不会因为用户轻微滚动而频繁重放。
>
> 这条规则**高于所有风格偏好** —— 用户说"我想要动效多"也不行,SOUL 底线是"读得清 > 动效多"。

## 视觉亮点(非动效,但跟动效协同)

> 用户反馈 v1.1.4:`你这默认效果太单调朴素了`。
> 回应:除了动效骨架外,在视觉锚点上做了增强。两件事协同:
> 1. **动效骨架**(本文件 2+5 模型)— 时间维度的丰富
> 2. **视觉锚点**(各 template 静态加强)— 空间维度的丰富

### 各 template 视觉锚点清单

| Style | 视觉亮点 |

### 主 Agent 出 v1 时查视觉锚点的决策

- template 已自带锚点 → 不主动加额外元素
- 用户特殊要求(摄影师/建筑师等)→ 加专属 visual mark(占位图 + alt 文案)
- 不要把所有锚点都开 → 选 3-5 个跟用户气质匹配的启用
