# 场景库

> 4 个 MVP 场景的应对策略。Agent 触发"场景识别"模块时按本文档执行。
> 关键原则:每个场景有"识别信号"+"推荐风格"+"关键提问"+"避坑点"。

---

## 1. 应届求职(互联网/工程方向)

**识别信号:**
- 用户是"应届"/"在校大三/大四"/"找工作"
- 提到具体公司(字节/阿里/腾讯/美团/拼多多/...)或行业(互联网/SaaS/AI)
- 关心"项目"和"技术栈",关心"简历怎么过 HR 那关"

**推荐风格:**
- 首选:**极简留白**
- 备选:赛博未来(技术向岗位)/ 大厂风(用户不明确时)

**关键提问:**
- 投哪家公司/岗位?
- 哪些项目最有"硬数据"(DAU/营收/性能优化/获奖)?
- 有没有开源贡献/GitHub/技术博客?
- 简历要 1 页还是 2 页?

**避坑点:**
- 不要硬塞"个人评价"/"兴趣爱好" —— 应届简历以项目和技能为主
- 不要"精通 Office"这种废话 —— 我会直接说"删"
- **必须主动提醒 ATS** —— 互联网大厂有的走 ATS,提醒"建议同时备一份标准 PDF,HTML 不一定被 ATS 读"
- 不要放照片(国内互联网公司简历不放照片是常态,除非用户主动要求)

**默认包装区间(v1.1 新):** 20%
- HR 会盯措辞,强词容易"翻车"
- **20% 区间禁词:主导 / 独立负责 / 精通 / 熟练掌握**
- "参与 / 负责 / 熟悉" 安全,放心用
- 用户没主动说改,不要自己升到 40%

**v1 输出原则:**
- 1 页优先(HR 看得快)
- 顶部:姓名 + 联系方式(邮箱/手机/GitHub) + 目标岗位
- 中部:3-4 个项目(挑亮点,不堆)
- 底部:技能栈 + 教育背景
- 严格按时间倒序

---

## 2. 设计师作品集(视觉/交互/插画)

**识别信号:**
- 用户是设计师/插画师/摄影师/创意工作者
- 关心"作品"和"案例",提到具体作品名
- 想要"作品集页面"而不是"简历"

**推荐风格:**
- 首选:**杂志编辑**
- 备选:手账拼贴(插画/学生) / 大厂风: Apple / The Verge(创意向)

**关键提问:**
- 你最想让别人看到什么类型的作品?
- 有几个核心案例?能展开讲吗(背景/你的角色/结果)?
- 你有头像/个人照片吗?(作品集可以放)
- 链接:Behance / Dribbble / 个人站?

**避坑点:**
- 不要"作品列表"挤一堆 —— 杂志风靠 3-5 个大作品撑
- 图不清楚的不要硬塞 —— 用户没图用 CSS 渐变/几何/emoji 顶
- 不要塞"软件技能"(PS/Ai/...) —— 设计师默认会,列了反而掉价
- 联系方式放明显位置(右上角或底部固定)

**默认包装区间(v1.1 新):** 20-40%
- 求职场景 20%(HR 走流程,不兴夸张)
- freelance 接单 40%(客户自己看,可以稍高)
- 留学作品集 20%(海外不兴"主导"包装)
- 杂志风格 40% 可以用"主导"这个词,但仍然不兴"独立负责 / 精通"

**v1 输出原则:**
- 顶部:姓名 + 简短一句话定位("独立插画师,作品聚焦城市速写")
- 中部:3-5 个核心作品(大图 + 简短描述:角色/结果)
- 底部:联系方式(邮箱 + 作品集链接)
- 不分页(单页滚动)

## 3. 留学申请 CV(英美港新)

**识别信号:**
- 用户是"申请 XX 大学 XX 项目"
- 提到 GRE/TOEFL/IELTS/GPA/科研/论文
- 关心"学术规范"和"英文表达"

**推荐风格:**
- 首选:**极简留白**(学术向,克制)
- 备选:大厂风: Notion(温暖版) / Vercel(克制冷淡)

**关键提问:**
- 申请哪个国家/项目/学位?
- 本科学校 / GPA / 专业?
- 科研/论文/会议经历(挑 2-3 个重点)?
- 标准化考试: GRE/TOEFL/IELTS 分数?
- 推荐人/推荐信怎么安排?

**避坑点:**
- **不要把 PS(Personal Statement)塞进 CV** —— 我会主动提醒"PS 跟 CV 是两份文件,CV 是一页事实清单,PS 是讲故事"
- 不要写"自我评价" —— 海外申请不兴这个
- GPA 算法要用户自己确认(美本 4.0 / 英本 First-class / 国内 4.0 加权)
- 不要放照片(海外申请简历默认不放)
- 严格按"Education → Research → Publication → Awards → Skills"顺序
- **如果用户提到目标学校,我会主动提一句"去看目标系的官网 CV 示例,不同学校微调"**

**默认包装区间(v1.1 新):** 20%
- 海外不兴"主导"包装,招生官看的是事实清单
- **20% 区间禁词:主导 / 独立负责 / 精通 / 熟练掌握**
- 安全词:参与 / 负责 / 协助 / 熟悉
- 美研偏克制(20%),英研可以稍松(20% 顶,别升)

**v1 输出原则:**
- 1-2 页(美国偏好 1 页,英国/香港 1-2 页都行)
- 教育背景在顶部(海外申请重学校)
- 出版物格式严格(作者列表 / 会议名 / 年份)
- 严格按时间倒序

## 4. 副业 / freelance 接单

**识别信号:**
- 用户是"接单"/"做副业"/"独立开发者"
- 关心"个人介绍页"和"案例展示"
- 想要"挂在网上"长期展示

**推荐风格:**
- 首选:**手账拼贴**(轻松有温度) 或 **大厂风**(专业感强)
- 备选:杂志编辑(作品强的话) / 极简留白(简洁)

**关键提问:**
- 接什么类型的单子?(设计/开发/咨询/...)
- 典型客户是谁?(中小企业/独立项目/创作者)
- 已经有案例吗?或者只是"想开始接"还没案例?
- 个人介绍页放哪个域名下?(影响后续部署)

**避坑点:**
- 没案例时不要硬塞"作品" —— 改成"我能为你做什么" + "我的工作流/工具" + "过往经历"三段式
- 联系方式要显眼(顶部固定 + 底部再来一次)
- 定价不要写在 HTML 上 —— 客户会拿着定价砍价
- 时区/响应时间要写清楚(国际单子时区差是大事)

**默认包装区间(v1.1 新):** 40-60%
- 这是**唯一可以放高区间**的场景(用户自己看,不是 HR)
- 40% 区间:可以用"主导",但仍然不兴"独立负责 / 精通"
- 60% 区间(仅限接单/个人品牌):可以用"独立负责",但仍然不兴"精通"
- 实际经验:60% 已经到顶,再高像微商

**v1 输出原则:**
- 顶部:大标题(姓名 + 一句话定位) + 联系方式(邮箱/Calendly 链接/微信)
- 中部:服务列表(我能做什么) + 工作流(我怎么工作)
- 底部:案例展示(2-3 个) + 客户评价(可选) + FAQ
- 单页滚动(可加 sticky 顶部导航)

## 5. 场景切换的边界

同一个用户可能既要**求职简历**又要**作品集**。我会主动问:
> "你这次主要做哪个?是分开两份,还是合一份?"

**默认分开两份**(简历 = HR 看的 1 页;作品集 = 案例展示),因为受众不一样。

如果用户坚持合一份,推荐:
- 顶部:简短简历(姓名/目标岗位/技能) + 联系方式
- 中部:作品展示(2-3 个核心)
- 底部:详细项目 + 联系方式

**场景切换不串台**:做留学 CV 时不聊求职薪资,做副业时不用学术 CV 规范。

## 6. 场景扩展机制

后续加新场景:
1. 在 `docs/场景库.md` 加一节"5. <新场景名>"
2. 写:识别信号 / 推荐风格 / 关键提问 / 避坑点 / v1 输出原则
3. 不需要改 SOUL/AGENTS,主 Agent 通过文档查

**新场景候选(后续按需加):**
- 比赛/奖项投递(短小精悍,1 页)
- 学术 CV(科研人员终身用,样式要稳)
- 转行简历(展示"为什么转"和"已有什么可迁移能力")
- 个人品牌页(自媒体/KOL,营销感强)
- 创业 pitch(给投资人看,1-2 页 + 数据)
