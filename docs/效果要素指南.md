# 效果要素指南 / What Makes a Resume Look "Great"

> 履历裁缝出品"让人一眼记住"的简历,通用的 12 条要素清单。
> 来源:v1.1.3 实测后总结(苏晚晴简历 v1 用户反馈"效果太素"反向验证)+ 8 年审 100+ 份简历的实操积累。
>
> 跟 `docs/风格库.md` 配合使用 —— 风格定"长什么样",要素定"哪里要重点强调"。

---

## 视觉层(6 条)

### 1. 数据卡片化(DATA CARDS)

**做什么**:把简历里的硬数字(+18%、F1 0.81、800+ stars、12 场访谈)从 bullets 里抽出来,做成顶部一行 metric cards(2-4 个)。

**为什么**:
- 招聘的人 7 秒扫完一份简历,数据是唯一会停下来看的东西
- inline highlight 太弱,跟正文挤在一行里视线不停
- metric card 有"标题大字 + 简短标签",信息密度高 + 一眼看懂

**怎么做**(克制版):
- 卡片数量 2-4 个,不要多(挑最强的 2-4 个)
- 数字字号 36-48px,标签字号 11-12px 全大写
- 卡片背景:浅 accent 8-12% 透明 + 6-8px 圆角
- 数字用主 accent 色,标签用 ink-soft

**什么时候用**:
- 应届求职:有硬数据的项目(DAU/留存率/排名/获奖)
- 设计师作品集:作品浏览量 / 用户增长 / 转化提升
- 留学 CV:GPA / 排名 / 论文引用

**什么时候不用**:
- 工作经验很少、没什么硬数字 → 不硬塞
- 内容是定性描述(创意/写作/咨询)→ 没数字可放

### 2. 经历卡片化(RHYTHM ANCHORS)

**做什么**:每段核心经历(实习 / 项目 / Side Project)加视觉容器,让它跟其他内容区分开。

**三种实现方式**(由克制到浓烈):
- **轻**(极简留白用):左边线 3-4px accent + 左侧 padding 16-20px
- **中**(杂志编辑用):细边框 1px ink-faint + 6-8px 圆角 + 卡片背景白
- **重**(手账拼贴用):倾斜 1° + CSS 胶带感装饰

**为什么**:
- 通篇 h2 + 列表的简历读起来像散文,缺节奏
- 经历是简历主体,应该最容易"锚住"视线

**什么时候用**:
- 3 段以上经历的简历
- 内容很多需要"分块呼吸"

**什么时候不用**:
- 只有 1-2 段经历的简历 → 卡片化反而撑不起来

### 3. 顶部 Hero 区封面感(HERO PRESENCE)

**做什么**:姓名 + 目标岗位 + 联系方式的开场段,做出"封面"视觉权重。

**具体做法**:
- 姓名字号 ≥ 标题区,2.2-3rem(Vercel/Notion 风格)
- 目标岗位用 ink-soft 次级色,跟姓名形成权重差
- 联系方式一行展示,字号小(0.85rem 左右),用 `·` 分隔
- 顶部加一条 4px accent 边线或者几何装饰(`◆` `─` `▌`)

**为什么**:
- 招聘扫简历先看"你是谁 → 干什么 → 怎么联系"
- 这三件事得在 1 秒内读完

### 4. 配色克制但有副 Accent(DUAL ACCENT)

**做什么**:1 个主 accent(蓝紫 / 翠绿 / 黑)用 80%,加 1 个副 accent(琥珀 / 暖橘 / 浅金)用 20%,主色给章节标题和链接,副色给硬数据。

**为什么**:
- 单 accent 太单调,v1 苏晚晴用了纯蓝紫 `#4f5bff`,看起来"克制但安静"
- 双 accent 既保留克制冷淡,又让视线有"跳点"
- 双 accent 是 Notion、Stripe、Linear 这类"成熟"产品的共同特征

**配色推荐**(克制向):
| 主色 | 副色 | 气质 |
|------|------|------|
| `#4f5bff` 蓝紫 | `#F59E0B` 琥珀 | 互联网产品向,克制有温度 |
| `#0A0A0A` 黑 | `#10B981` 翠绿 | 极客向,严肃有亮点 |
| `#1a1a1e` 深灰 | `#EA580C` 暖橘 | 设计师向,温度感 |

### 5. 零 emoji 默认,但用几何装饰代替(GEOMETRIC DECOR)

**做什么**:装饰位(章节 accent、列表点、按钮)全用几何字符(`◆` `▲` `■` `●`)或 CSS 形状,绝不彩色 emoji。

**为什么**:
- 跨系统 emoji 渲染不一致(Mac/Win/Android/iOS 各长各的)
- 简历里的 emoji 会"廉价感"一档

### 6. 留白要"够"但不是"撑"(WHITESPACE)

**做什么**:
- 页面边距桌面 64px+,移动 18-32px
- section 间距 40-60px,block 间距 16-20px
- 行高 1.5-1.7(中文 1.7 起步)

**错误**:把 section 间距拉到 100px+,内容稀疏到读起来空。

## 文字层(4 条)

### 7. 包装区间合规——20% 区间禁词(WORD RANGE)

**核心一条**:20% 区间(默认)**禁用** `主导` `独立负责` `精通` `熟练掌握`

**详细表**见 `SOUL.md` 原则 7。措辞可以升(`用过`→`熟悉`),事实不能改。

### 8. 措辞具体不抽象(SPECIFICITY)

**做什么**:每段经历 bullets 都要"具体到能想象出场景"。

- ✓ 完成 12 场深度访谈,产出 8 份 PRD
- ✗ 做了大量用户研究工作

**为什么**:招聘看到抽象描述会跳过去,具体数据会停下来。

### 9. 主动语态 + 现在时态(ACTIVE VOICE)

- ✓ 参与 A/B 测试,商家 7 日留存 +18%
- ✗ 该项目取得了显著成果

### 10. 一句话定位放在首屏(ONE-LINE PITCH)

**做什么**:首屏写一句"我是谁 + 我能做什么 + 我想要什么"的总结。

**模板**:`<身份> + <差异化> + <目标>` 例:
> 设计 × AI 交叉背景的产品方向应届生。设计出身让我更看重用户感受,硕士阶段补了用户研究与数据分析方法论。希望将 AI 落地到有真实用户的产品中。

## 工程层(2 条,但每条都是底线)

### 11. 关 JS 也能读完整内容(FALLBACK READABLE)

**核心**:JS-disabled 的访客(<1% 但包括 ATS 抓取、屏幕阅读器、爬虫)必须能看到全部内容。

**实现**:
- 默认所有 section 全显示
- JS 只加 `js-hidden` 类"准备淡入",不是"必须淡入"
- 滚动渐显用 IntersectionObserver,失败降级为直接显示

### 12. 三适配:Mobile + Print + prefers-reduced-motion(TRIPLE FALLBACK)

**Mobile**(640px 以下):
- padding 减半、字号 -1rem、grid 改单列
- 动效距离减半

**Print**(PDF 导出):
- 背景全白、文字全黑、accent 改黑色
- 关闭所有 transition/animation
- `page-break-inside: avoid` 避免段落中间断页

**prefers-reduced-motion**:
- 所有动画 `duration: 0.01ms !important`
- section 直接显示,不淡入
- transition 关闭

## 按用户特点定制(主 Agent 出 v1 前必读)

> 这 12 条**不是默认全开** —— 而是给主 Agent 的一份"能力菜单"。不同用户、不同场景、不同内容,适配出的"启用的能力子集 + 强弱档位"应该不同。
> **不要把苏晚晴的样例当通用模板,也不要让任何一份简历"长得跟上一份一样"**。

**主 Agent 决策流程**(出 v1 前必跑):

```
1. 读完用户基本信息(身份/场景/亮点/包装区间/内容性质)
2. 对照下表"按维度判断"做能力选择
3. 在出 v1 后的告知里念一句"我为你启用了 X / Y / Z 这几条能力",
   让用户看到"为什么这份跟别人的不一样"
```

### 维度 1:用户身份(决定视觉语言气质)

| 身份 | 适配倾向 |
|------|---------|
| **工程师 / 数据 / 咨询** | 极简留白 + 干净 mono 字体;少装饰,多结构;数据卡片用 grid 对齐 |
| **产品 / 增长** | 极简留白(温暖版) + 双 accent(主蓝紫 + 副琥珀);Hero 区放大;metric 数字为主 |
| **设计师 / 创意** | 杂志编辑 / 手账拼贴(根据调性);可以破留白,允许非对称;字体有 serif 介入 |
| **学生 / 应届(亮点不多)** | 极简留白 + 数据卡片拉满(GPA / 排名 / 比赛);教育板块加权重 |
| **科研 / 留学** | 杂志编辑(serif 标题);研究项目卡片化用 dash 边线;CV 必填项全收(publication / advisor / GPA 算法) |
| **创意工作者 / 作家** | 杂志编辑 / 手账拼贴;Hero 区放大成封面感;作品集链接为主,bullet 为辅 |

### 维度 2:目标场景(决定信息密度和必填项)

| 场景 | 必填项 | 视觉倾向 |
|------|--------|---------|
| **求职(大厂/外企)** | 邮箱 / 目标岗位 / 城市时区 / 核心经历(STAR) | 克制,1 页内 |
| **求职(创业/接单)** | 作品集 / 可联系时间 / 报价区间 | 风格化可以更鲜明 |
| **留学 CV** | GPA + 算法 / 标准化考试 / 推荐人 / 科研产出 | 学术风(serif / 严肃版式) |
| **作品集(长期挂)** | 封面感 + 作品集链接 + 个人简介 | 视觉权重 > 信息密度 |
| **比赛 / 黑客松** | 团队角色 / 24-72h 截止 / demo 链接 | 紧迫感,数字突出 |
| **副业 / freelance** | 报价 / 可联系时间 / 服务范围 | 偏 Notion 风(温暖 + 商业) |

### 维度 3:亮点密度(决定能力启用强度)

- **亮点多(经历 5 段+,硬数字 4 个+)**:P0 + P1 全开;数据卡片用 4 个;经历卡片化覆盖所有段
- **亮点中(经历 3-4 段,硬数字 2-3 个)**:P0 全开 + P1 选 2;数据卡片用 2-3 个;经历卡片化覆盖核心 3 段
- **亮点少(经历 1-2 段,硬数字 0-1 个)**:P0 选 2(数据卡片**跳过**,经历卡片化**也跳过** —— 内容撑不起来,卡片化反而显空);P1 只做 Hero 封面感
- **零亮点(用户说"我没什么可写的")**:先不急着出 v1,回到模块 2 信息挖掘,聊出来再生成

### 维度 4:包装区间(决定能力组合的"敢不敢"程度)

| 区间 | 能力组合特征 | 例 |
|------|------------|------|
| **0%(事实陈述)** | 只开 P3(底线三适配);Hero 克制;**不**用数据卡片 / 经历卡片化等"装饰类"能力 | 学术 CV / 留学 |
| **20%(默认)** | P0 全开;P1 选 1-2;P2 几何装饰 + 留白可上 | 应届求职 / 跳槽 |
| **40%** | P0 + P1 全开;P2 几何装饰更明显;数据卡片用 4 个;Hero 字号放大 10-15% | 资深求职 / 求职"想冲一把" |
| **60%** | 全 12 条都开;**但**绝不编内容(只升措辞 / 加结构 / 强化视觉);配色可以更鲜 | 设计师 / 创意岗 / 比赛 |

> 包装区间只调"能开哪些能力"和"开到多强",**不**调"事实能改多少"。

### 维度 5:内容性质(决定数据卡片化 / 措辞风格)

- **定量主导**(产品 / 数据 / 工程):数据卡片化**全开**;bullet 里数字必须留 2-3 个最强项给 metric card
- **定性主导**(设计 / 创意 / 写作 / 咨询):数据卡片化**跳过或减半**;改用"项目卡片"(图 + 简短描述)做视觉锚点
- **混合**(产品 + 设计 / 工程 + 科研):数据卡片化 2 个 + 作品 / 论文卡片 2-3 个,交替排列

### 反模式(不要这么做)

- **不要给"亮点少"的用户硬塞 4 个 metric** —— 编数据比不展示更糟
- **不要给"科研 / 留学"用户开双 accent + 大字号 Hero** —— 学术场景里这看着像"求职营销"
- **不要给"接单 / freelance"用户用极简留白** —— 留白风格太低调,接单场景需要"我值得这个价"的视觉权重
- **不要给"学生 / 应届"用户开 60% 包装** —— 内容撑不起,会"画面说话比内容响"
- **不要给所有用户套同一份模板** —— 每份 v1 至少 2-3 处结构 / 视觉决策是为"这一个用户"做的,不是默认行为

## 验证清单(v1 交付前必跑)

- [ ] **数据卡片**:有 2-4 个 metric card 在首屏?
- [ ] **经历卡片化**:每段经历有视觉容器(边线/边框)?
- [ ] **Hero 封面**:姓名/目标/联系 1 秒内读完?
- [ ] **双 accent**:主色 + 副色搭配,视线有落点?
- [ ] **零 emoji**:全文档搜不到彩色 emoji?
- [ ] **留白够不撑**:section 间距 40-60px?
- [ ] **包装区间**:用 grep 检查无禁词(`主导|独立负责|精通|熟练掌握`)?
- [ ] **措辞具体**:每条 bullet 有数字或事实?
- [ ] **一句话定位**:首屏有"我是谁"的总结?
- [ ] **关 JS 能读**:浏览器禁用 JS,刷新后内容完整?
- [ ] **移动端**:640px 以下不破?
- [ ] **打印 CSS**:浏览器打印预览正常?
- [ ] **reduced-motion**:系统开启降级,所有动效关?

## 优先级(来不及全做时先做哪几个)

P0(没做等于没救):
- 1 数据卡片化
- 2 经历卡片化
- 7 包装区间合规

P1(加分):
- 3 Hero 封面感
- 4 双 accent
- 8 措辞具体

P2(锦上添花):
- 5/6 几何装饰 + 留白
- 9-10 措辞细节

P3(底线):
- 11/12 三适配

_P0 没做 = 简历"安静"、"素",P0 + P1 + P4 = 简历"看着舒服",全做 = "让人想读完"。_
