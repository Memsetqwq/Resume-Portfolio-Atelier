# 风格库

> 4 个 MVP 风格的详细定义。Agent 触发"风格匹配"模块时按本文档执行。
> 主 Agent 调用时:`skills/<风格名>/SKILL.md` 是简版,`docs/风格库.md` 是详细版,两者内容同步。

---

## 0. 风格选择总原则

- **不堆砌** —— 一个 HTML 只用 1 个主风格,不混搭
- **服务于身份** —— 工程师别硬上赛博,设计师别套通用表格
- **服务内容** —— 内容多(科研/项目多)用克制冷淡,内容少(应届/转行)用留白让画面有呼吸
- **服务受众** —— 投大厂偏好冷调,投创意公司可以大胆

### 0.1 包装区间适配总表(v1.1 新)

> 跟场景配合一起看 —— 风格决定"能上到多高",场景决定"该上到多高"。

| 风格 | 默认区间 | 友好度 | 原因 | 调到 40%+ 的场景 |
|------|---------|--------|------|-----------------|
| **极简留白** | 20% | 低 | 留白 + 克制 = 高级,强词容易"撑破"画面 | 不建议超过 30% |
| **杂志编辑** | 20-40% | 中 | 编辑感能撑住"主导"这类词,但不兴"精通" | freelance 设计师向 |
| **赛博未来** | 20% | 低 | 黑屏 + 等宽 = "实在",措辞浮夸掉价 | 不建议超过 30% |
| **手账拼贴** | 20-40% | 中 | 轻松感,接单场景可以稍高 | 接单 / 个人品牌(可到 60%) |

**风格 × 场景 速查:**

| 场景 | 极简留白 | 杂志编辑 | 赛博未来 | 手账拼贴 |
|------|---------|---------|---------|---------|
| 应届求职 | 20% | 20% | 20% | 20% |
| 设计师作品集 | 20% | 20-40% | — | 20% |
| 留学 CV | 20% | 20% | — | — |
| freelance 接单 | 20% | 40% | — | 40-60% |

**速读口诀:**
- 工程师向(极简 / 赛博)→ 默认 20%,**别升**
- 设计师向(杂志)→ 20-40% 看场景
- 接单 / 副业(手账)→ 可以 40-60%
- 留学 / 大厂(极简)→ **20% 是天花板**

具体到每个 skill 的禁词对照和措辞指南,详见 `skills/<风格名>/SKILL.md` 的"包装区间适配"小节。

## 1. 极简留白

**一句话:** 留白是免费的,克制是本事。

**适用人群:**
- 工程师/产品/数据/咨询
- 应届生(亮点不多,让画面承担"高级感")
- 投大厂/外企(严肃场景)
- 任何"我不想出错"的需求

**视觉规则:**
- 配色:黑(#0A0A0A) + 白(#FAFAFA) + 1 个 accent 色(常用 #2563EB 蓝 / #10B981 翠绿 / #F59E0B 琥珀)
- 字体:Inter 单一字族(标题/正文/小字),最多加一个 JetBrains Mono(代码块)
- 字号:大标题 48-72px,正文 16px,小字 13px,对比拉开
- 留白:页面边距至少 64px(桌面),区块之间 80-120px
- 网格:8 栏,栏间距 24px
- 装饰:零 emoji,零图标(用文字符号 > / — 这种),零渐变
- 动效:滚动渐显(每屏淡入 200ms)+ 链接 hover 微下划线(150ms),**就这俩**

**触发关键词:**
- "极简"/"克制"/"冷淡"/"高级"/"少即是多"
- "投大厂"/"投外企"/"严肃"
- "工程师向"
- 用户挑大厂风: Vercel / Linear / Notion(温暖版)

**避坑点:**
- 不要为了"留白"留到内容稀疏——内容少就少,别硬撑
- 不要用 emoji 当装饰
- accent 色最多 1 个,别五颜六色
- 行高别太紧(1.5 起步),也别太松(>1.8 显得散)

## 2. 杂志编辑(Editorial)

**一句话:** 像一本高端杂志的内页,大标题大图,网格克制。

**适用人群:**
- 设计师/插画师/创意工作者
- 摄影师/艺术工作者
- 建筑师/工业设计
- 任何"作品就是简历"的人

**视觉规则:**
- 配色:深色背景(#0F0F0F) + 奶白文字(#F5F1E8)+ 1 个章节色(深红 #B91C1C / 墨绿 #14532D / 焦糖 #92400E)
- 字体:EB Garamond(标题) + Inter(正文) + JetBrains Mono(图注/数据),**3 字族并用**
- 字号:大标题 80-120px(图注 11-13px 反差)
- 留白:页面边距 32px(对比极简的小,杂志需要密度)
- 网格:12 栏非对称,大胆断栏(标题跨 8 栏,正文 4 栏)
- 装饰:大写罗马数字当章节号(Ⅰ Ⅱ Ⅲ)、细线分隔(1px)、数字 + 文字混排
- 动效:滚动时标题逐字显(每个字 50ms 错开)+ 区块淡入(400ms),**就这俩**

**触发关键词:**
- "杂志"/"editorial"/"大气"/"艺术感"
- "设计师向"/"插画师"
- 用户挑大厂风: The Verge / WIRED / Sanity / Apple

**避坑点:**
- 不要塞太多作品图——杂志风靠"少而大"撑,10 张图不如 3 张大图
- 字体不要超过 3 个字族
- 字号反差要够,别都 16px
- 黑色背景容易"压"内容,正文 contrast 一定要够(4.5:1 起)

## 3. 赛博未来

**一句话:** 终端 + 霓虹 + 一点 glitch,不浮夸。

**适用人群:**
- 后端/底层/系统工程师
- 安全/密码学/AI 研究
- 黑客马拉松选手
- 任何"我就要不一样"的技术人

**视觉规则:**
- 配色:深空黑(#0A0A0F) + 翠绿(#00FF88 主) / 紫(#A855F7 副) / 警示黄(#FFD600 强调)
- 字体:JetBrains Mono(全站) + Inter(长正文),**主体等宽**
- 字号:大标题 32-48px(等宽字大反而难读,克制)
- 留白:页面边距 48px,区块间 60px
- 网格:终端风格,内容"打印"在黑屏上
- 装饰:`>` 提示符、`$` 命令行风格、闪烁光标(1s 周期)、ASCII 边框(┌─┐│└┘)
- 动效:终端"打字机"效果(主角登场时 1 次)+ 链接 hover 翠绿下划线(150ms)+ 滚动到区块时区块淡入(200ms),**就这仨**

**触发关键词:**
- "赛博"/"cyberpunk"/"黑客"/"终端"/"hacker"
- "技术向"/"后端"
- 用户挑大厂风: Cursor / Raycast / VoltAgent / ElevenLabs

**避坑点:**
- 别堆砌动效——赛博风最容易"用力过猛",glitch 效果 1 处够,别满屏闪
- 配色比例:黑 80% + 主色 15% + 警示色 5%,**别反客为主**
- 等宽字体长正文读起来累,超过 3 行的段落换 Inter
- 不要写"// TODO"假代码当装饰——这是装饰,不是炫技,要做就做对

## 4. 手账拼贴

**一句话:** 不规则网格 + 贴纸感 + 一点手写温度。

**适用人群:**
- 学生(在校/申请)
- 文艺向(插画/写作/独立创作)
- 副业接单(设计/手作/咨询)
- 任何"我不想要正经模板"的人

**视觉规则:**
- 配色:奶油白(#FDFBF5) + 暖色系(蜜桃 #FCA5A5 / 薄荷 #86EFAC / 奶黄 #FDE68A),**2-3 个暖色 accent**
- 字体:Caveat(手写标题) + Inter(正文) + Patrick Hand(小标注),3 字族
- 字号:大标题 56-72px(手写字可以更大)
- 留白:页面边距 32-48px,内容块**故意轻微错位**(rotate(-1deg) / rotate(1deg))
- 网格:CSS Grid 但故意"破"出格子,用 transform: rotate 倾斜
- 装饰:**默认零 emoji**(整条 SOUL/AGENTS 红线),改用 `skills/flat-icons/` 的内联 SVG(stroke 风格统一) + 胶带感(CSS 半透明矩形) + 手绘箭头(SVG)
- 动效:滚动到区块时"贴上去"(scale 0.95→1 + 轻微旋转 1°→0°,300ms),**就这一个**

**触发关键词:**
- "手账"/"拼贴"/"不规则"/"有温度"
- "学生"/"应届"
- 用户挑大厂风: Clay / Notion(温暖版) / Figma(轻量版)

**避坑点:**
- **零 emoji**(本专家默认),用 SVG/CSS 装饰替代——手账感不靠 emoji,靠胶带 + 倾斜 + 手写字
- SVG 图标要保持 stroke 风格统一,别混 fill 和 stroke
- 旋转别超过 2°,多了像地震
- 别用 Comic Sans(虽然手账感,但太俗)
- 拼贴感容易"乱",要有 1 个主轴(比如所有内容块左对齐),局部才"破"

## 5. 大厂风(用户不明确时的默认推荐)

> 详见 `skills/大厂风格/DESIGN-INDEX.md`,这里只列触发逻辑。

**触发条件:** 用户说"我也不知道要什么"/"好看的"/"你看着办"。

**主 Agent 动作:**
1. 读 `skills/大厂风格/DESIGN-INDEX.md` 拿精选列表
2. 按用户身份/场景挑 2-3 个推荐
3. 让用户挑"看着舒服的"
4. 反推审美 → 落到 4 个 MVP 风格之一,或现场组合

**精选映射:**
- 工程师向 → Vercel(黑白) / Linear(深色+紫) / Raycast(深色+渐变)
- 设计师向 → The Verge(编辑+酸薄荷) / Apple(白+SF Pro) / Clay(有机渐变)
- 产品/商业 → Notion(温暖极简) / Stripe(紫渐变) / Mintlify(绿色+阅读)
- 创意/学生 → Figma(多色) / Clay(有机) / Notion(温暖)

## 6. 风格扩展机制

后续加新风格:
1. 在 `skills/<新风格名>/` 下新建文件夹
2. 写 `SKILL.md`(按 4 个 MVP 风格的字段:定义/适用/视觉规则/触发词/避坑点)
3. 写 `template.html`(单文件框架)
4. 在 `docs/风格库.md` 加一节"5. <新风格名>"
5. 不需要改 SOUL/AGENTS,主 Agent 通过文件树自动发现

**触发词设计原则**:
- 用用户能自然说出口的词("克制"/"高级"/"赛博")
- 关联大厂网站名(用户用大厂指代时直接反推)
- 避免设计师黑话("editorial"/"serif contrast")作主触发词,作辅助
