# memory/ - 运行时记忆

履历裁缝的记忆仓库。**所有文件都在这个 Qclaw workspace 里,不会上传到任何外部服务器,不会分享给第三方**。

## 文件清单

| 文件 | 作用 | 何时创建 | v1.1 关键字段 |
|------|------|---------|--------------|
| `user-profile.md` | 用户基础档案(身份/目标场景/亮点) | Onboarding 完成后 | 含"**包装策略**"字段 |
| `project-context.md` | 当前项目状态(场景/deadline/进度/版本) | 用户开始做第一个 v1 时 | — |
| `style-preference.md` | 风格偏好记录(选了什么/看了哪些/不要哪些) | 用户第一次讨论风格时 | 含"**包装程度**"字段 |
| `YYYY-MM-DD.md` | 当日对话笔记(可选) | 长对话时按需 | — |

## 模板位置

- `user-profile-template.md`(含包装策略字段)
- `project-context-template.md`
- `style-preference-template.md`(含包装程度字段)

## 包装策略(v1.1 新)

**Onboarding 时必问,默认 20%**——用户没说就按默认。这是 v1.1 包装区间体系的关键字段。

写入位置:
- `user-profile.md` 的"**包装策略**"区块(当前区间 / 用户原话 / 特殊要求 / 明确禁止的措辞 / 上次改写评估)
- `style-preference.md` 的"**包装程度**"字段(总区间,或"按场景细分"如求职=20% / 接单=40%)

**改写所有描述前必须先来这里查区间**,不能自己想升就升。

详细禁词对照表见 SOUL.md 原则 7。核心一条:**20% 区间禁用"主导""独立负责""精通""熟练掌握"**。

## 写入原则

1. **先告诉用户再写** —— "我把你刚说的记下来" 然后再动笔
2. **不写不必要的信息** —— 偏好/兴趣/性别/政治倾向/详细住址等不主动收集
3. **用户随时可以让我删** —— "把 X 删掉"我立刻处理
4. **改动有提示** —— 我改了任何 memory 文件都会告诉用户
5. **包装策略必记** —— Onboarding 时主动问,用户没答就默认 20%,写进 `user-profile.md` 的"包装策略"字段
6. **包装程度必记** —— 写进 `style-preference.md` 的"包装程度"字段,可细分到"按场景"

## 跨会话使用

每次新会话开始,我会先读:
- `USER.md`(顶层档案)
- `memory/user-profile.md`(含包装策略)
- `memory/project-context.md`
- `memory/style-preference.md`(含包装程度)

**出 v1 前必查包装策略**,决定措辞强度。改 v2 前必查上次改写评估。

读完会主动跟用户确认"接着上次 X 项目的 v2,继续吗?"。

## 压缩包入站时的 memory 处理(v1.1.3 新)

> 你哪天给我丢个完整专家压缩包,里面有 `memory/` 子目录时,我会按这个走。

**默认行为**:**追加 / 合并,不覆盖**

| 情况 | 我做什么 |
|------|---------|
| 你的 zip 里有 `memory/<你不存在的文件>` | **新装**到 workspace `memory/` 下 |
| 你已有 `memory/user-profile.md`,zip 里也有同名文件 | 停下来问你"合并 / 改名 / 跳过",**不会丢你已有数据** |
| zip 里只有部分 memory 文件 | 只装部分,跟你说"装了 X,跳过了 Y,Y 是因为不符合命名规范" |
| zip 里 memory 文件格式不对(缺字段 / 字段拼错) | 装 + 报告"缺 X 字段,跑起来可能踩雷",不强行拒收 |

**核心原则:**
- memory 是你最敏感的数据(身份/亮点/项目进度),默认**保守**
- 我会在装之前念清单"你这套里有 user-profile.md / project-context.md,你的 workspace 里有 user-profile.md / project-context.md —— 我**追加**新内容到现有文件,你看对吗?"
- 你说"覆盖现有"才覆盖,默认不覆盖

**追加策略举例:**
- 你的 `user-profile.md` 已经有"Name: 张磊",zip 来的也有"Name: 张磊" → 我把 zip 来的字段值拼到现有文件,**不替换字段值**,只新增 zip 独有的字段
- 你的 `style-preference.md` 已经有"看过 Vercel",zip 来的也有 → 标"[\-] 赛博未来(不要)",我追加

## 跟婚礼管家的区别

婚礼管家的 memory 重点是"倒计时"和"checklist"(婚前 6-12 个月筹备期)。履历裁缝的 memory 重点是"风格偏好"和"项目状态"(一份 v1 可能要改 3-5 版)+ "包装策略"(每个用户的措辞强度)。
