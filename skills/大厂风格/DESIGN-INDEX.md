## 工程师向(投大厂/技术岗)

### 1. Vercel
- **链接**:https://getdesign.md/vercel/design-md
- **气质**: 黑白精确、Geist 字体、零装饰
- **适合**: 工程师求职简历(前端/全栈/平台)
- **推荐理由**: 黑白精确 = 工程师"靠谱"气质,无装饰 = 内容即简历
- **反推方向**: 极简留白 + Inter 字体 + 0 渐变
- **包装友好度**: 低(默认 20%,不兴强词)

### 2. Linear
- **链接**:https://getdesign.md/linear.app/design-md
- **气质**: 深色+紫、超精准、超极简
- **适合**: 后端/平台/AI 工程师
- **推荐理由**: 深色 = 不一样的"克制",紫 accent = 高级感
- **反推方向**: 极简留白(深色版) + 紫色 accent
- **包装友好度**: 低(默认 20%,克制气质禁强词)

### 3. Raycast
- **链接**:https://getdesign.md/raycast/design-md

- **适合**: 极客向工程师(命令行/工具/效率)
- **推荐理由**: 渐变 accent 让"克制"不死板
- **反推方向**: 极简留白 + 渐变 accent
- **包装友好度**: 低(默认 20%)

## 反推逻辑(主 Agent 怎么用)

用户挑了 1-2 个"看着舒服的"后,主 Agent 这样做:

1. **用户挑 1 个**: 直接读这个大厂的完整 DESIGN.md(getdesign.md 上的链接),提取配色/字体/网格,套到 `skills/<MVP 风格名>/` 模板上
2. **用户挑 2 个**:"取 A 的 X + B 的 Y"——比如"Vercel 的黑白 + Notion 的衬线标题"——现场组合,落到最接近的 MVP 风格,或临时改一个混合版
3. **用户挑 3 个以上**: 主 Agent 反推共性("你挑的都偏克制+冷调"),落到 1 个 MVP 风格,跟用户确认

**不 1:1 复刻** — 借的是"视觉语言"(克制/温暖/编辑感),不是抄设计(配色/字体的具体值)。版权是底线。
