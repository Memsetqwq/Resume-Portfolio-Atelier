# 大厂风格 / Brand-style recommendations

> 当用户说"我也不知道要什么"时,主 Agent 触发"大厂风推荐"流程,读本目录的内容。

---

## 这是什么

我手头有一批大厂公开的"设计系统"——不是凭印象,是逐家分析过的视觉规则(配色/字体/排版/网格/动效)。

用大厂做"启动器"的好处:
- 用户不用懂设计词汇,用"看哪个网站舒服"代替
- 大厂风格经过市场验证,不容易踩"小众审美"的雷
- 反推出来比直接问"你想要什么"准得多

来源: [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md)(MIT License) — 73 个大厂的 DESIGN.md 公开设计系统。

## 怎么用

主 Agent 触发"大厂风推荐"流程时:
1. 读 `DESIGN-INDEX.md` 拿精选列表
2. 按用户身份/场景挑 2-3 个推荐
3. 让用户挑"看着舒服的"
4. 反推审美 → 落到 `skills/<MVP 风格名>/`,或现场组合

**完整 73 个大厂 DESIGN.md** 不在本地仓里(避免冗余),用户可以自己访问 https://getdesign.md/ 看任意一家。

## 文件结构

- `README.md` — 本文件
- `DESIGN-INDEX.md` — 精选 10-15 个适合简历/作品集的大厂,按身份/场景分类

## 边界

**大厂风推荐是"启动器",不是"终点"**。用户挑完大厂后可能会说"我喜欢 Vercel 的克制,但想要更温暖一点"——这种"取大厂一半,再往前走一步"的需求才是我做长项。

我不会做 1:1 复刻某个大厂的视觉(版权风险),只"借气质"——比如"Vercel 的黑白精确 + Notion 的温暖衬线标题"。

## 包装区间联动(v1.1 新)

> 大厂风格落到 MVP 风格时,要把包装区间一并考虑,不要等落完风格才校准。

**反推到 MVP 风格时的默认区间:**

| 落到 | 默认区间 | 友好度 | 来源大厂举例 |
|------|---------|--------|--------------|
| 极简留白 | 20% | 低 | Vercel / Linear / Mintlify / Stripe |
| 杂志编辑 | 20-40% | 中 | The Verge / WIRED / Sanity / Apple |
| 赛博未来 | 20% | 低 | Cursor / VoltAgent / Raycast |
| 手账拼贴 | 20-40%(接单可 60%) | 中 | Clay / Figma / Notion(暖版) |

主 Agent 出 v1 前必查 `memory/user-profile.md` 的"包装策略"字段,确认区间后再反推风格。**没查到 → 默认 20%**。

**具体禁词对照表**(20% 区间):主导 / 独立负责 / 精通 / 熟练掌握 —— 全禁。详细见 SOUL.md 原则 7。

## 引用规范

每个精选大厂条目都注明来源链接,用户想深入看时点开即可。如果用户问"这是不是抄 X 网站",我会说明"借的是视觉语言,不是抄设计"。
